iOS V.5.0.5 - 03/02/18

Property Details

  • This page was having a lot of memory leaks and performance issue. We rebuilt this page from the ground up, and that has increased the load time by x4

Web Views

  • Web views (search form, mortgage calc, area stats, account info) loading was a big wonky in that it would only load once the user got to those sections. 
  • Property details web views are now loaded in the background once the page is in view, so the UX is a bit more fluid.
  • We fixed performance issues with the search form loading and managing states on the app

Property Photo Gallery

  • We were having memory leaks within the app (especially on iPad) and we traced this back to issues with a library we were using for the photo gallery. We rebuilt the gallery ourselves to be able to have control on how it loads images and manages memory. This change has VASTLY increased performance of the app.
  • We now lazy load one image back from the one the user is viewing to help with UX and load time
  • Fallback image for photos was updated

Agent List

  • This list loading has always been a networking and memory sync when we have loaded it, so we changed it to an autocomplete. 
  • This also helps with agents being sensitive about not being at the top of the list
  • We are now able to load agent photos on this view without it being a memory hog

Agent Details

  • This page was having a lot of performance issues, so we rebuilt it all together with a news design. This should be a VAST improvement from a UX standpoint as well as app performance

Map Search

  • We found some bugs with the app's search protocol being executed twice in some situations. We have done a fair amount of clean up to howe this executes to increase performance.

App API StyleSheet

  • We have updated the app to offload almost all configs to a stylesheet API endpoint. This will give us even more control over text, colors and app configurations
